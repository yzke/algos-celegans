# ALGOS Phase 1.0 设计稿
# 图本位神经系统:从矩阵范式到图范式

> 项目阶段:从 Phase 0 探索期进入 Phase 1 架构期
> 终极目标:数字生命
> 当前里程碑:秀丽虫数字版作为可行性验证

---

## 0. 设计原则

本设计建立在 Phase 0-0.9a 八个阶段的经验教训上。以下原则是非可妥协的基础:

0.1 图作为 ground truth

神经系统的本体是图。节点是神经元,边是突触连接。
矩阵不是底层结构,是子图在某个时刻的局部计算视图。

0.2 接近生物的纯逻辑抽象

只保留对动力学有因果效应的对象。所有"中间机制"——
钙浓度、髓鞘、特定离子通道——被抽象为节点属性、边属性
或节点处理函数的内部细节,不作为独立变量存在。

0.3 计算和学习同层

不存在"训练时刻"和"推理时刻"的分离。某些参数有更新规则,
每帧执行;它们更新是图的演化的一部分,跟状态 V 的更新同等地位。

0.4 子图分解显式存在

整个 connectome 被分解为多个功能子图(回路)。每个子图对应
一类计算——感觉处理、命令决策、运动生成等。子图在每帧被
实时矩阵化,运算完写回主图。

0.5 时间通过事件传播,不通过同步更新

信号在图上的传播是事件驱动的,带显式延迟。一个神经元只在
有信号到达时才被激活;没有信号时它休眠(不浪费计算)。

0.6 抽象层级保持单一

不引入跨抽象层级的混杂。神经元层只关心信号变换,
不关心其分子实现;调质层(如果有)只关心全局状态,
不关心其化学实现。

## 1. 数据结构

### 1.1 图

Graph:
  nodes: 节点集合
  edges: 边集合
  subgraphs: 子图集合(对子集 nodes 和 edges 的引用)
图是 first-class 对象。所有运算都通过图接口完成。

### 1.2 节点(神经元)

Node:
  # 静态属性(初始化设定,不变)
  id: 唯一标识符(例如 'ASEL', 'AVAL')
  category: 'sensory' | 'inter' | 'motor' | 'pharyngeal' | 'modulator'
  neurotransmitter: 'glutamate' | 'GABA' | 'acetylcholine' | 'neuropeptide' | ...
  is_plastic: 是否参与可塑性
  
  # 动态属性(每帧可能变化)
  V: 当前激活值
  internal_state: 节点的内部记忆(适应变量、过去事件等)
  
  # 处理参数(默认 frozen,某些情况下可标记为 plastic)
  threshold: 激活阈值
  tau: 时间常数
  f: 处理函数(可以是统一的,也可以每节点不同)
### 1.3 边(突触)

Edge:
  # 静态属性
  source: 源节点
  target: 目标节点
  type: 'chemical' | 'electrical' | 'modulatory'
  
  # 动态属性
  weight: 权重(可能 plastic 也可能 frozen)
  delay: 信号传播延迟(以 tick 计)
  sign: 兴奋(+1) 或 抑制(-1)
  
  # 学习相关
  is_plastic: 是否可塑
  plasticity_rule: 如果可塑,用什么规则更新
### 1.4 子图

Subgraph:
  name: 例如 'forward_command', 'chemotaxis_loop', 'feeding_CPG'
  type: 'feedforward' | 'recurrent'
  nodes: 这个子图包含的节点引用
  edges: 这个子图包含的边引用
  
  # 计算相关
  state: 这个子图的内部状态(如果是 recurrent 类型)
  matrix_view: 实时从图提取的矩阵视图(缓存)

## 2. 节点的处理函数 f

每个节点收到输入,执行一个变换,产生输出。

### 2.1 标准节点函数

最简形式(对应 LIF 神经元的逻辑核心):

对每帧 t:
  incoming_signals = 所有恰好在 t 到达节点的信号
  net_input = 加权求和(incoming_signals)
  
  V(t) = V(t-1) × (1 - 1/tau) + net_input
  
  if V(t) > threshold:
    输出信号 = 1
    V(t) = 0  # 重置
  else:
    输出信号 = 0
这是"积分-发放"模型的本质。信号是离散事件(0 或 1),
神经元积累输入直到阈值,然后发出 spike 并重置。

### 2.2 特殊节点

某些神经元(主要是感觉神经元、调质神经元)有不同的 f。
这些差异通过节点的 category 和参数表达,不需要为每个
节点写独立代码。

感觉节点: f 接收外部输入(从环境或 mock 数据),
按某种方式编码为内部 V,然后按标准规则发放。

调质节点: f 与标准节点类似,但它的输出不是发往特定
后继节点,而是更新全局调质浓度。调质浓度通过特殊"调质边"
影响其他节点。

## 3. 边的信号传播

### 3.1 标准化学突触

当源节点 i 在时刻 t 发出 spike:
  在时刻 (t + delay[i,j]) 信号到达节点 j
  到达时的信号值 = sign[i,j] × weight[i,j]
  这个值被加入节点 j 的 incoming_signals 队列
延迟是显式的,跨 tick 的信号通过 history buffer 维护。

### 3.2 电突触(gap junction)

电突触不传 spike,传膜电位本身,几乎瞬时。

对于电耦合边 (i, j):
  V_i 和 V_j 之间存在持续的双向拉力
  在每帧的状态更新中,加入项 weight[i,j] × (V_j - V_i) 到 i 的输入
  对称地加入 weight[i,j] × (V_i - V_j) 到 j 的输入
电突触是少数需要"同步更新"而不是"事件驱动"的机制。

### 3.3 调质边

调质边 (调质节点 m → 目标节点 i):
  调质节点的活动累积为浓度 c_m(t)
  浓度通过调质边影响节点 i 的某些参数
  影响形式由调质类型决定(例如:乘性 gain 调制 threshold)
调质的特殊性:
- 不传递 spike,传递的是慢变浓度
- 影响是参数级的(改变 threshold/tau/weight),不是状态级的
- 时间常数远长于普通突触(几百到几千 tick)

## 4. 子图分解

### 4.1 单向子图(feedforward)

特征:信号有明确的入口和出口,中间没有反向连接。

例子(基于秀丽虫已知回路):
- 触碰反射: ALM/AVM → AVD → 后退运动神经元
- 化学感觉前向: ASE → AIY → AIZ → ...

数学上:节点可以拓扑排序。

执行方式:每帧,信号沿子图的拓扑顺序前向传播一步。

### 4.2 环形子图(recurrent)

特征:存在反馈连接,信号自维持。

例子:
- 命令电路: AVA ↔ RIM ↔ AVE 等的相互连接
- 头部运动 CPG: RMD 神经元间的振荡
- 整合环路: AIY ↔ AIZ ↔ RIA 等的多向连接

数学上:邻接矩阵存在闭环。

执行方式:每帧推进一步,环路状态跨帧维持。这种子图
天然产生持续活动,不需要外部输入也能维持(像 CPG 一样)。

### 4.3 子图的识别

来源(按优先级):

优先级 1: 已知生物回路

从文献整理的明确回路:
- 命令电路(forward/backward)
- 触碰反射弧
- 化学感觉处理通路
- 头部运动 CPG
- 摄食 CPG
- 主要调质回路(RID, 5-HT, AVK)

优先级 2: 图论方法补全

对剩余神经元和连接,用社区检测算法(例如 Louvain)
找出强连接子区域,作为候选子图。

优先级 3: 行为聚类(留到后期)

如果数据足够,基于真虫子的活动模式聚类找子图。
Phase 1.0 不做这一步。

### 4.4 重叠子图

一个神经元可以属于多个子图。例如 AVA 同时属于:
- 后退命令电路
- 触碰反射(作为命令端)
- 某些感觉处理回路的末端

这种重叠是必要的,不是设计缺陷。它意味着:
- 共享节点是子图间的耦合点
- 共享节点的状态在所有它所属的子图中同步可见
- 不同子图通过共享节点交换信息

## 5. 每帧的计算流程

`
每帧 t:
1. 收集所有"在 t 到达"的信号
     - 来自上一帧的 spike,根据延迟计算到达时刻
     - 来自感觉输入(从环境或 mock 数据)
  
  2. 对每个子图,实时提取矩阵视图
     - 单向子图:对应的稀疏前向矩阵
     - 环形子图:对应的稀疏递归矩阵
  
  3. 在子图内执行矩阵运算
     - 单向:net_input = W_feedforward @ active_signals
     - 环形:state_new = W_recurrent @ state_old + new_input
  
  4. 每个节点应用 f 函数
     - 用步骤 3 的 net_input 和节点的 V_history
     - 产生新的 V 或 spike
  
  5. 写回图
     - 节点的 V 和 internal_state 更新
     - 子图的 state 更新(如果是 recurrent)
  
  6. 调质浓度更新
     - 慢动力学:c_m(t+1) = c_m(t) + (target - c_m(t))/tau_m
  
  7. 可塑性更新(只对 plastic 边)
     - 应用 Hebbian 或 STDP 规则
     - W[i,j] 微调一点点
  
  8. 信号入队
     - 产生 spike 的节点,其输出信号入队
     - 入队时计算到达时间 = t + delay
  
  9. 检查死亡条件(如果接入肉体)
     - 暂时不在 Phase 1.0 内
---

## 6. 学习机制

### 6.1 哪些参数可塑

**默认所有参数 frozen**。只在文献明确支持可塑性的地方,
把对应边的 weight 标记为 plastic。

秀丽虫已知可塑的边大约 50-100 条,涉及神经元:
- AWC (化学感觉学习)
- AIY, AIB, AIZ (关联学习)
- RIA, RIM (整合)
- AFD (温度记忆)
- PLM, AVD (机械习惯化)

具体清单从 Phase 0.8 已经整理过,这次延用。

### 6.2 学习规则

最简形式:Hebbian 规则,带衰减。

对每条 plastic 边 (i → j):
  ΔW = η × pre.V × post.V - λ × W
  W_new = W + ΔW
  W_new 被夹在 [W_min, W_max] 范围内

参数:
- η (学习率): 小常数,例如 1e-4
- λ (衰减): 更小,例如 1e-5
- W_min, W_max: 防止权重发散

### 6.3 学习的时间尺度

可塑性效果在 10^3 到 10^4 tick 后才能显著观察到。
这跟生物的"几分钟到几小时学习"对应。

不需要专门的"训练阶段"。学习随网络运行持续发生。

## 7. 调质系统

### 7.1 调质作为图中的节点

调质生产者(如 RID, NSM, RIC)是图中的普通节点,
属性是 `category = 'modulator'`。

调质的"输出"不通过普通化学突触,而通过"调质边"
连接到所有受它影响的节点。

### 7.2 调质浓度

每个调质类型有一个全局浓度 c_m(t),由它的生产节点活动驱动:

target_m = mean(V of producer_nodes)
c_m(t+1) = c_m(t) + (target_m - c_m(t)) / tau_m

tau_m 远大于普通节点的 tau(几百 tick),反映调质扩散和清除的慢动力学。

### 7.3 调质对其他节点的影响

通过调质边,c_m 影响目标节点的参数:

对受调质影响的节点 i:
  effective_threshold = threshold_base × (1 + sum over modulators: 
                                          c_m × modulator_sensitivity[i, m])

具体哪些节点对哪些调质敏感,从文献整理。Phase 0.8 已经
建立了初步的敏感度数据。

### 7.4 调质在 Phase 1.0 的范围

Phase 1.0 包含调质机制的基础设施(数据结构、计算流程),
但不一定激活所有调质类型。

最小配置:实现 RID 类(神经肽前进促进)作为示例。
后续可以增加其他调质类型。

## 8. 时间和帧率

### 8.1 一个 tick 的语义

不规定 tick 对应的物理时间。系统活在自己的时间里。

需要的时间尺度通过相对比例表达:
- 神经元 tau: 5-50 tick
- 调质 tau: 500-2000 tick
- 学习显现: 10^3-10^4 tick
- 验证实验长度: 10^4-10^5 tick

### 8.2 主循环

while not terminated:
    execute_one_frame(graph)
    tick += 1
    
    if tick % MODULATOR_UPDATE_RATE == 0:
        update_modulators(graph)
    
    if tick % EVALUATION_INTERVAL == 0:
        log_metrics()
`

## 9. 验证策略

### 9.1 内部验证

- 数值稳定性:跑 10^5 tick 无 NaN,V 不发散
- 子图正确性:子图内信号流符合预期(例如触碰反射弧能触发后退)
- 调质效应:调质浓度能影响目标节点的兴奋性

### 9.2 外部验证

复用 Phase 0.7 的三个指标,跟 Atanas 2023 真虫子数据对比:
- subspace_alignment
- temporal_correlation
- fc_similarity

目标不是马上进入真虫子分布(我们知道当前指标远未达到),
而是看新架构相对 Phase 0.9 的提升幅度。

### 9.3 对比基线

跟 Phase 0.9 (CTRNN + 同质 + 单调质) 比较:
- 在哪些指标上改善
- 在哪些神经元类别上改善最明显
- 反相关(Phase 0.8 诊断的核心缺失)能否产生

如果新架构在反相关产生能力上显著改善,即使三个指标
没大幅提升,也算架构验证成功。

## 11. 关键风险

### 11.1 工程风险

新架构工作量较大(估 1-2 周)。可能出现:
- 图实现的隐藏 bug,难调试
- 子图分解粒度不对(分得太粗或太细)
- 性能问题(纯 Python 图遍历慢)

### 11.2 理论风险

H_3 也可能错。可能的情况:
- 反相关不来自子图切换,来自更深的机制
- 现有图论分解方法找不到合适的功能子图
- 事件驱动的稀疏激活在 302 节点规模上没有优势

如果 H_3 错,我们需要再次修正假设。但每次修正都更精确,
这是项目方法论的常态。

### 11.3 范围风险

数字生命是终极目标。秀丽虫是阶段验证。需要避免:
- 把秀丽虫做得过度精确,变成"另一个 OpenWorm"
- 把架构做得过度通用,失去秀丽虫验证价值
- 用秀丽虫的特殊性(只有 302 节点)做出无法拓展的设计

Phase 1.0 的目标是:架构在秀丽虫规模上跑通,但设计上为
更大规模的数字生命留好接口。

## 12. 这个设计跟现有研究的区别

### 12.1 跟 OpenWorm 比

OpenWorm: 精确物理仿真,微分方程 + 分子动力学。
我们: 功能逻辑抽象,纯数学图运算。

OpenWorm 在分子-物理层有不可替代的价值。
我们在功能-计算层提供互补视角。

### 12.2 跟传统 SNN 研究比

传统 SNN: 一个大的稀疏网络,所有神经元同等地位,
可能预定义层次。

我们: 显式的功能子图分解,子图间通过共享节点交互,
学习作为图演化的内置机制。
### 12.3 跟 GNN 比

GNN: 图作为输入数据,矩阵作为网络层。
我们: 图作为网络本身,矩阵作为子图的局部视图。

### 12.4 跟当前 LLM 比

LLM: 单一大型同质网络,训练时学习,推理时锁死。
我们: 异质功能子图组合,学习持续发生,跟"使用"同层。

## 13. 后续阶段展望

Phase 1.0 完成后,可能的路径:

### 13.1 Phase 1.5: 肉体接入

把肉体设计为另一个图(感觉器官子图 + 运动器官子图),
通过桥接边连接到神经系统主图。

整个系统(神经 + 肉体 + 环境)成为同一个大图的不同区域。

### 13.2 Phase 2: 自然演化

让图本身可以演化:
- 边的增删(突触形成/修剪)
- 节点的增删(新生神经元)
- 拓扑的动态重组

这一步开始接近真正的"数字生命"。

### 13.3 Phase 3: 跨物种拓展

把秀丽虫验证过的架构原则,应用到更大的系统:
- 更多神经元
- 更复杂的子图结构
- 更丰富的可塑性

最终目标:架构本身成为一种通用的"数字生命操作系统",
可以承载从秀丽虫规模到大脑规模的各种神经系统。

---

## 14. 概念总结

把整个设计压成最简:

神经系统 = 图
计算 = 子图内的矩阵运算
学习 = 图中某些参数的可塑性
时间 = 事件在图上的传播

每一帧:
- 信号在图上沿边按 delay 传播
- 子图被实时矩阵化
- 矩阵运算产生新的节点状态
- 状态写回图
- 可塑性更新某些边权重
- 调质浓度按慢动力学演化

没有训练阶段。没有推理阶段。只有持续运行的图。
图的存在和演化本身,就是这个数字神经系统的"生命"。
